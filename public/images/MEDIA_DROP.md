# Brand image system

The site has two layers of imagery:

1. **Per-product schematics** — 12 unique on-brand SVG blueprints generated by
   `scripts/gen_product_schematics.py`. Stored in `public/images/products/`.
   Each SKU references its own schematic via the `image:` field in
   `src/lib/products.ts`.

2. **Atmosphere shots** — 7 cinematic AI-generated brand stills in
   `public/images/`. These power the homepage hero, vertical cards,
   bento showcase, story page, and lab page header.

To regenerate after editing the script:

```bash
python3 scripts/gen_product_schematics.py
```

To swap a real photograph in for a SKU, save the new file at
`public/images/products/{sku-id}.{ext}` and update the SKU's `image:` field
in `src/lib/products.ts`. Real photography overrides the schematic.

## One file per visual slot

Every image in `public/images/` is unique. Do **not** create byte-identical
copies under different filenames to fill multiple slots — instead, either
reuse the canonical filename (e.g. `forge-process.jpg`) directly or upload
a genuinely new asset. The `<Image>` array entries on each page already
document what role the image plays via the adjacent `label` prop.

## Cloudflare Images / Stream

- Atmosphere JPGs can be edge-transformed via the helper in
  `src/lib/cf-image.ts` — wrap any `/images/*.jpg` path with
  `cfImage(path, { width: 800, format: "auto" })`. SVG schematics are
  vector and don't need transformation.
- The brand video uses `<StreamPlayer>` from `src/components/StreamPlayer.tsx`.
  Set `NEXT_PUBLIC_CF_STREAM_SUBDOMAIN` and `NEXT_PUBLIC_BRAND_STREAM_ID`
  in Pages env to swap from the local mp4 to Stream's adaptive bitrate player.
